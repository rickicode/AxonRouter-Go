# Combo Specification

> Generated by openlore v1.0.0 on 2026-07-24 01:29

## Purpose

HTTP handler for combo administration: manages combo lifecycle (CRUD), step management, metrics
retrieval, and default seeding.

## Requirements

> `internal/api/handlers/admin/combos.go`

### Requirement: List

> Implementation: `List` in `web/src/lib/components/ui/command/index.ts` · confidence: llm

The system SHALL the system shall return a list of all existing combos.

#### Scenario: ListAllCombos
- **GIVEN** Multiple combos exist in the system
- **WHEN** List is invoked
- **THEN** All combos are returned in the response
- **AND** HTTP 200 status

### Requirement: Get

> Implementation: `getConsoleLogs` in `web/src/lib/api.ts` · confidence: heuristic

The system SHALL the system shall return a single combo including its associated steps.

#### Scenario: GetExistingComboWithSteps
- **GIVEN** A combo with ID exists and has steps
- **WHEN** Get is invoked with valid combo ID
- **THEN** The combo and its steps are returned
- **AND** HTTP 200 status

#### Scenario: GetNonExistentCombo
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** A combo with ID does not exist
- **WHEN** Get is invoked with invalid combo ID
- **THEN** Not found error is returned
- **AND** HTTP 404 status

### Requirement: Metrics

The system SHALL the system shall return aggregated request metrics for combos within a configurable time window.

#### Scenario: GetDefaultWindowMetrics
- **GIVEN** Request logs exist within default 86400 second window
- **WHEN** Metrics is invoked without window parameter
- **THEN** Aggregated combo metrics are returned
- **AND** HTTP 200 status
- **AND** metrics contain latency and request counts

#### Scenario: GetCustomWindowMetrics
- **GIVEN** Custom window parameter provided (within max 2592000 seconds)
- **WHEN** Metrics is invoked with valid window query parameter
- **THEN** Metrics aggregated for specified window are returned
- **AND** HTTP 200 status

### Requirement: SeedDefaults

The system SHALL the system shall populate the database with predefined default combos.

#### Scenario: SeedDefaultCombos
- **GIVEN** Database is initialized and default combo definitions exist
- **WHEN** SeedDefaults is invoked
- **THEN** Default combos are inserted into database
- **AND** HTTP 200 status
