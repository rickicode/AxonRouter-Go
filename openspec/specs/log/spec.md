# Log Specification

> Generated by openlore v1.0.0 on 2026-07-24 01:29

## Purpose

Handles HTTP requests for request log querying, retrieval, cleanup, real-time active request
monitoring, and aggregated usage statistics for administrative dashboard.

## Requirements

> `internal/api/handlers/admin/logs.go`

### Requirement: ListRequestLogs

> Implementation: `List` in `web/src/lib/components/ui/command/index.ts` · confidence: llm

The system SHALL returns paginated request logs with optional filters for provider, connection, model, combo, modality, status, date range, and search terms.

#### Scenario: SuccessfulPaginatedQueryWithFilters
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** Multiple request logs exist in the database
- **WHEN** List is called with valid page, per_page, and filter parameters
- **THEN** Returns paginated results matching all specified filter criteria
- **AND** legacy 'provider_type_id' parameter is accepted as alias for 'provider_id'

#### Scenario: DateRangeFilteringWithISODates
- **GIVEN** Request logs span multiple days
- **WHEN** List is called with start_date and end_date parameters
- **THEN** Returns only logs within the specified date range
- **AND** legacy 'since' parameter accepted as milliseconds timestamp

#### Scenario: FilterByStatusCode
- **GIVEN** Request logs exist with various status codes
- **WHEN** List is called with status_code parameter
- **THEN** Returns only logs matching the specified HTTP status code
- **AND** legacy 'until' parameter accepted as milliseconds timestamp

### Requirement: ClearOldRequestLogs

> Implementation: `logs` in `web/src/lib/stores.ts` · confidence: heuristic

The system SHALL deletes request logs older than a specified retention window of 7, 30, or 90 days.

#### Scenario: SuccessfulDeletionWithValidRetentionPeriod
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** Request logs older than 30 days exist in the database
- **WHEN** Clear is called with older_than_days: 30
- **THEN** Deletes all request logs with timestamp older than 30 days and returns count of deleted records
- **AND** Only values 7, 30, 90 are permitted

#### Scenario: RejectionOfInvalidRetentionPeriod
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** Any request body with invalid retention period
- **WHEN** Clear is called with older_than_days: 15
- **THEN** Returns 400 Bad Request with validation error
- **AND** Error message indicates invalid value

### Requirement: GetActiveInFlightRequests

The system SHALL returns currently in-flight proxied requests for real-time monitoring on the logs page dashboard.

#### Scenario: ListActiveRequestsDuringTraffic
- **GIVEN** Multiple requests are currently being proxied through the system
- **WHEN** ActiveRequests is called
- **THEN** Returns all in-flight requests with their current state
- **AND** Data sourced from active request tracking module

#### Scenario: EmptyActiveRequestsDuringIdle
- **GIVEN** No requests are currently being proxied
- **WHEN** ActiveRequests is called
- **THEN** Returns empty array
- **AND** Returns empty array when no active requests

### Requirement: GetSingleLogEntry

The system SHALL retrieves a single request log entry by its unique identifier with all detailed fields including token breakdown, latency, cost, and client information.

#### Scenario: SuccessfulRetrievalOfExistingLog
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A request log with ID 'abc123' exists in the database
- **WHEN** Get is called with id: 'abc123'
- **THEN** Returns complete log record with all fields populated
- **AND** Nullable fields use pointer types in response

#### Scenario: LogNotFound
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** No request log with ID 'nonexistent' exists
- **WHEN** Get is called with id: 'nonexistent'
- **THEN** Returns 404 Not Found with error message
- **AND** Error indicates log not found

### Requirement: GetAggregatedLogStatistics

> Implementation: `logs` in `web/src/lib/stores.ts` · confidence: heuristic

The system SHALL returns aggregated usage statistics across providers, models, and daily breakdown for dashboard analytics.

#### Scenario: DefaultStatisticsForLast24Hours
- **GIVEN** Request logs exist for multiple providers and models over past 24 hours
- **WHEN** Stats is called without hours parameter
- **THEN** Returns provider usage, model usage for 24 hours, and daily usage for 30 days
- **AND** Daily usage always covers 30 days regardless of hours parameter

#### Scenario: CustomHoursWindowForStatistics
- **GIVEN** Request logs exist for the past 72 hours
- **WHEN** Stats is called with hours: 72
- **THEN** Returns provider and model usage for 72 hours, daily usage for 30 days
- **AND** Allows custom time window for provider and model aggregations

> `internal/api/handlers/admin/console_logs.go`

### Requirement: GetConsoleLogs

> Implementation: `getConsoleLogs` in `web/src/lib/api.ts` · confidence: heuristic

The system SHALL reads the console log file from the end and returns up to maxconsolelines lines as a json response.

#### Scenario: SuccessfulLogRetrieval
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** Console log file exists with content
- **WHEN** Get is invoked via HTTP request
- **THEN** Returns up to 500 lines from end of file in reverse chronological order
- **AND** Path is returned in response

#### Scenario: MissingLogFile
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** Console log file does not exist
- **WHEN** Get is invoked via HTTP request
- **THEN** Returns empty lines array without error
- **AND** Path is returned in response

#### Scenario: UnreadableLogFile
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** Console log file exists but is unreadable
- **WHEN** Get is invoked via HTTP request
- **THEN** Returns HTTP 500 with error message
- **AND** Error message is returned in JSON


## Sub-components

> `ConsoleLogsHandler` is an orchestrator. Each sub-component below implements one logical block.

### Sub-component: OpenLogFile

> Implements: `os.Open`

Opens the console log file for subsequent read operations.

#### Requirement: OpenLogFile

The system SHALL open the console log file at the configured path for read access.

#### Scenario: SuccessfulOpen
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** a valid log file path is configured
- **WHEN** opening the log file
- **THEN** SHALL return a non-nil file handle with no error
- **AND** log file exists at configured path

#### Scenario: MissingFile
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** the configured log file path does not exist
- **WHEN** opening the log file
- **THEN** SHALL return an error
- **AND** log file does not exist

### Sub-component: CheckFileNotExist

> Implements: `os.IsNotExist`

Determines if a file operation failed because the target file does not exist.

#### Requirement: CheckNotExist

The system SHALL determine whether an error indicates that a file does not exist.

#### Scenario: NotExistError
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** an error from a file operation
- **WHEN** checking if error indicates missing file
- **THEN** SHALL return true
- **AND** error is os.ErrNotExist

#### Scenario: OtherError
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** an error from a file operation
- **WHEN** checking if error indicates missing file
- **THEN** SHALL return false
- **AND** error is permission denied

### Sub-component: CloseFileHandle

> Implements: `f.Close`

Releases system resources held by an open file handle.

#### Requirement: CloseFile

The system SHALL release all resources associated with the open file handle.

#### Scenario: SuccessfulClose
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** a valid open file handle
- **WHEN** closing the file handle
- **THEN** SHALL return nil error and release resources
- **AND** file was successfully opened

#### Scenario: DoubleClose
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a file handle that has been closed
- **WHEN** closing the file handle again
- **THEN** SHALL return an error
- **AND** file already closed

### Sub-component: GetFileStats

> Implements: `f.Stat`

Retrieves metadata about the open log file to determine its size.

#### Requirement: RetrieveFileInfo

The system SHALL retrieve metadata about the open file including size and modification time.

#### Scenario: SuccessfulStat
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** a valid open file handle
- **WHEN** querying file statistics
- **THEN** SHALL return non-nil FileInfo with accurate size
- **AND** file is open and readable

#### Scenario: InvalidHandle
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a closed or invalid file handle
- **WHEN** querying file statistics
- **THEN** SHALL return an error
- **AND** file handle is invalid

### Sub-component: GetFileSize

> Implements: `stats.Size`

Extracts the byte size of the log file to calculate read offset.

#### Requirement: RetrieveSize

The system SHALL return the size in bytes of the file from its metadata.

#### Scenario: NonEmptyFile
- **GIVEN** a FileInfo object from a valid file
- **WHEN** querying file size
- **THEN** SHALL return 1024
- **AND** file contains 1024 bytes of log data

#### Scenario: EmptyFile
- **GIVEN** a FileInfo object from an empty file
- **WHEN** querying file size
- **THEN** SHALL return 0
- **AND** file is empty

### Sub-component: QueueLength

> Implements: `len`

Returns the current element count in the byte queue for capacity planning.

#### Requirement: GetQueueLength

The system SHALL return the number of elements currently stored in the byte queue.

#### Scenario: NonEmptyQueue
- **GIVEN** a byte queue with elements enqueued
- **WHEN** querying queue length
- **THEN** SHALL return 5
- **AND** queue contains 5 elements

#### Scenario: EmptyQueue
- **GIVEN** a newly initialized byte queue
- **WHEN** querying queue length
- **THEN** SHALL return 0
- **AND** queue is empty

### Sub-component: ConvertToInt64

> Implements: `int64`

Converts numeric values to int64 for offset calculations in file operations.

#### Requirement: ConvertInt64

The system SHALL convert a numeric value to a signed 64-bit integer.

#### Scenario: PositiveConversion
- **GIVEN** a non-negative integer value
- **WHEN** converting to int64
- **THEN** SHALL return int64(1024)
- **AND** value is 1024

#### Scenario: NegativeConversion
- **GIVEN** a negative integer value
- **WHEN** converting to int64
- **THEN** SHALL return int64(-1)
- **AND** value is -1

### Sub-component: ReadAtOffset

> Implements: `f.ReadAt`

Reads log data from a specific byte offset in the file for tail operations.

#### Requirement: ReadFileAt

The system SHALL read bytes from the file starting at a specified offset into the provided buffer.

#### Scenario: SuccessfulRead
<!-- openlore-test: tags=smoke,regression (auto) -->
- **GIVEN** a valid open file and read buffer
- **WHEN** reading at offset 0
- **THEN** SHALL return 5 bytes read with 'hello' in buffer and nil error
- **AND** file contains 'hello world' at offset 0
- **AND** buffer size is 5

#### Scenario: OffsetBeyondEOF
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a valid open file
- **WHEN** reading beyond file end
- **THEN** SHALL return io.EOF error
- **AND** offset exceeds file size

### Sub-component: BytesToString

> Implements: `string`

Converts raw byte data read from the log file into a human-readable string.

#### Requirement: ConvertBytes

The system SHALL convert a byte slice to its string representation.

#### Scenario: ValidUTF8
- **GIVEN** a byte slice containing UTF-8 data
- **WHEN** converting bytes to string
- **THEN** SHALL return equivalent string
- **AND** bytes are valid UTF-8

#### Scenario: AsciiBytes
- **GIVEN** a byte slice with ASCII values
- **WHEN** converting bytes to string
- **THEN** SHALL return 'ABC'
- **AND** bytes contain [65, 66, 67]

### Sub-component: CloseStore

> Implements: `Close`

Releases database resources held by the Store when shutting down.

#### Requirement: CloseDatabase

The system SHALL close the underlying database connection and release associated resources.

#### Scenario: SuccessfulClose
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** a Store with an open database connection
- **WHEN** closing the store
- **THEN** SHALL return nil and close the connection
- **AND** database connection is active

#### Scenario: IdempotentClose
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a Store whose connection was previously closed
- **WHEN** closing the store again
- **THEN** SHALL return an error or nil based on implementation
- **AND** connection already closed
