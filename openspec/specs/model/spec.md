# Model Specification

> Generated by openlore v1.0.0 on 2026-07-24 01:29

## Purpose

Manages model capability metadata by loading embedded JSON configuration and providing thread-safe
lookup with wildcard suffix fallback for dated model variants.

## Requirements

### Requirement: GetCapabilities

The system SHALL returns capabilities for a given full model id with wildcard fallback to progressively shorter suffixes separated by '-'.

#### Scenario: ExactModelIDMatch
- **GIVEN** Model ID exists exactly in capsMap

- **THEN** Corresponding ModelCapabilities is returned
- **AND** No wildcard fallback is performed

#### Scenario: WildcardFallbackMatch

- **WHEN** GetCapabilities is called with a dated or suffixed model ID
- **THEN** Capabilities of the matching shorter suffix variant are returned
- **AND** Suffixes are progressively stripped from the rightmost '-'

#### Scenario: NoMatchFound
- **GIVEN** No exact or wildcard match exists for the model ID
- **WHEN** GetCapabilities is called with an unknown model ID
- **THEN** Empty ModelCapabilities is returned
- **AND** All possible suffixes have been attempted

#### Scenario: NoWildcardSeparator
