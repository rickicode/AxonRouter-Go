## Purpose

An AI API gateway and router that aggregates multiple LLM providers (OpenAI, Claude, Gemini, etc.)
behind a unified interface. It handles request translation between formats, authentication, caching,
rate limiting, and usage tracking, with a web dashboard for operational management.

## Requirements

### Requirement: SystemCapabilities

The system SHALL provide the following capabilities:
- Use an executor framework (BaseExecutor/OpenAIExecutor) to abstract provider-specific execution logic and enable unified streaming, retry, and failover handling
- Implement bidirectional translators between all major formats (OpenAI, Claude, Gemini, Antigravity) rather than forcing a single internal canonical format
- Employ file-backed settings with in-memory caching and sync.Map for high-concurrency connection state to avoid database write contention on hot routing paths
- Restrict sensitive admin operations from master key authentication as a defense-in-depth measure against key compromise
- Use singleflight deduplication for OAuth token refresh to prevent thundering herd against provider token endpoints
- Implement NDJSON streaming backup format with encryption support for database portability and disaster recovery
- Deploy edge relay functions to serverless platforms (Vercel, Deno, Cloudflare) for distributed proxy infrastructure

## Technical Notes

- **Architecture Style**: Layered architecture with clear separation between API handlers, business services, and infrastructure concerns. The system uses an executor framework for provider abstraction, with dedicated translators for cross-format request/response conversion. This layering enables provider-agnostic routing while maintaining format-specific endpoint compatibility.
- **Security Model**: Dual authentication system: (1) OAuth 2.0 for end-user provider connections with token refresh and singleflight deduplication, stored via CAS pattern; (2) Master admin API key for dashboard/admin API access with cryptographic randomness. Critical admin operations (password change, key regeneration, upgrade, restart, backup/restore, TLS config) are explicitly restricted from master key authentication to prevent lateral movement if the master key is compromised. Connection-scoped credential isolation.
- **External Integrations**: OpenAI API, Anthropic Claude API, Google Gemini API, Grok API, Cloudflare Workers AI, Vercel Edge Network, Deno Deploy, Cloudflare Workers, GitHub Releases (for upgrades), External DNS/IP discovery services, systemd (for service management)
