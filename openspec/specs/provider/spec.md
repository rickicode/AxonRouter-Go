# Provider Specification

> Generated by openlore v1.0.0 on 2026-07-24 01:29

## Purpose

Manages per-provider runtime configuration including routing mode selection and round-robin
indexing. Provides file-backed settings storage with in-memory caching and concurrent-safe access to
avoid database write contention on hot routing paths.

## Requirements

### Requirement: GetNextRoundRobinIndex

The system SHALL returns the next round-robin index for a provider-model combination, cycling through candidates. each providerid+modelid pair maintains an independent counter to prevent cross-model interference.

#### Scenario: FirstCallInitializesCounter
- **GIVEN** No prior round-robin calls for this provider-model pair
- **WHEN** NextRoundRobinIndex is called with total > 0
- **THEN** Returns 0 and initializes atomic counter for the key
- **AND** Counter initialized to 0
- **AND** Returns 0 on first call

#### Scenario: SequentialRotation
- **GIVEN** Existing counter for provider-model pair
- **WHEN** NextRoundRobinIndex is called repeatedly
- **THEN** Returns next index in cyclic sequence
- **AND** Counter wraps using modulo arithmetic
- **AND** Sequential indices returned: 0, 1, 2, ..., total-1, 0, 1, ...

#### Scenario: IsolatedModelCounters
- **GIVEN** Multiple provider-model pairs with active counters
- **WHEN** NextRoundRobinIndex is called for different provider-model pairs
- **THEN** Each pair returns independent sequence unaffected by others
- **AND** Each key maintains independent counter

- **AND** The restored selection SHALL match the user's original selection boundaries
- **AND** The visually highlighted text SHALL correspond to the original user selection

### Sub-component: SelectElementContents

> Implements: `selectElementText`

Programmatically selects all text content within a given DOM element for copying or visual emphasis.

#### Requirement: SelectAllElementText

The system SHALL the system shall select all text content within a specified html element by creating a range that encompasses the element's contents and adding it to the current selection.

#### Scenario: SelectElementTextContent
- **GIVEN** A valid HTMLElement exists in the document with text content
- **WHEN** The selectElementText function is called with the target element
- **THEN** All text within the specified element is selected in the document
- **AND** The element's text content SHALL be visually highlighted
- **AND** The document selection SHALL contain a single range spanning the element's contents

### Sub-component: CopyInfoFeedback

#### Scenario: InformFallbackCopy

- **THEN** An informational toast is displayed with guidance such as 'Press Ctrl+C to copy'

- **AND** The toast notification SHALL provide actionable guidance to the user
