# Usage Specification

> Generated by openlore v1.0.0 on 2026-07-24 01:29

## Purpose

Extracts and merges token usage counts and costs from streaming and non-streaming API responses
across multiple provider formats including OpenAI, Claude, Gemini, and Grok CLI. *(analyzed in 2
chunks)*

## Requirements

> `internal/api/handlers/v1/stream.go`

### Requirement: ExtractTokensFromBody

> Implementation: `Body` in `web/src/lib/components/ui/table/index.ts` · confidence: heuristic

The system SHALL extract token counts from a non-streaming response body supporting openai-format, native claude-format, gemini usagemetadata, and openai responses api usage shapes.

#### Scenario: OpenAIFormatWithPromptAndCompletionTokens
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A non-streaming response body in OpenAI format with usage.prompt_tokens and usage.completion_tokens
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with extracted values
- **AND** InputTokens set to prompt_tokens value
- **AND** OutputTokens set to completion_tokens value

#### Scenario: OpenAIFormatWithInputAndOutputTokens
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A non-streaming response body in OpenAI format with usage.input_tokens and usage.output_tokens
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with extracted values
- **AND** InputTokens set to input_tokens value
- **AND** OutputTokens set to output_tokens value

#### Scenario: OpenAIFormatWithCacheDetails
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A non-streaming response body with prompt_tokens_details containing cached_tokens and
cache_creation_tokens
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with cache information
- **AND** CachedTokens set to cached_tokens value
- **AND** CacheCreationTokens set to cache_creation_tokens value

#### Scenario: OpenAIFormatWithReasoningTokens
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A non-streaming response body with completion_tokens_details containing reasoning_tokens
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with reasoning information
- **AND** ReasoningTokens set to reasoning_tokens value

#### Scenario: NativeClaudeFormat
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A non-streaming response body in native Claude format with usage.input_tokens, usage.output_tokens,
usage.cache_creation_input_tokens, and usage.cache_read_input_tokens
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with aggregated input and cache information
- **AND** InputTokens is sum of input_tokens, cache_creation_input_tokens, and cache_read_input_tokens
- **AND** CachedTokens set to cache_read_input_tokens
- **AND** CacheCreationTokens set to cache_creation_input_tokens

#### Scenario: GeminiNonStreamingFormat
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A non-streaming response body in Gemini format with usageMetadata containing promptTokenCount and
candidatesTokenCount
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with extracted metadata
- **AND** InputTokens set to promptTokenCount
- **AND** OutputTokens set to candidatesTokenCount
- **AND** CachedTokens set to cachedContentTokenCount
- **AND** ReasoningTokens set to thoughtsTokenCount

#### Scenario: OpenAIResponsesAPIFormat
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A non-streaming response body in OpenAI Responses API format with response.usage.input_tokens and
response.usage.output_tokens
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with extracted values
- **AND** InputTokens set to input_tokens
- **AND** OutputTokens set to output_tokens

#### Scenario: UnrecognizedOrEmptyFormat
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A response body that does not match any known format or has no valid usage data
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts with zero values
- **AND** All fields remain at zero values

#### Scenario: FormatPrecedenceWithFallback
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** A response body that could potentially match multiple formats but has zero or invalid values in
earlier formats
- **WHEN** ExtractTokensFromBody is called
- **THEN** Returns StreamTokenCounts from the first format with positive input tokens
- **AND** OpenAI format is attempted first
- **AND** Claude format is attempted second

### Requirement: ExtractTokensFromSSEChunk

The system SHALL extract token counts from a server-sent events (sse) chunk supporting streaming formats including codex and openai responses api response.completed/done shapes.

#### Scenario: ValidSSEChunkWithTokenData
<!-- openlore-test: priority=high tags=smoke (auto) -->
- **GIVEN** An SSE chunk containing valid token usage data in a supported streaming format
- **WHEN** ExtractTokensFromSSEChunk is called
- **THEN** Returns StreamTokenCounts with extracted values and true
- **AND** Boolean return is true

#### Scenario: InvalidOrNonTokenSSEChunk
<!-- openlore-test: priority=high tags=regression (auto) -->
- **GIVEN** An SSE chunk that does not contain token usage data or is in an unsupported format
- **WHEN** ExtractTokensFromSSEChunk is called
- **THEN** Returns empty StreamTokenCounts and false
- **AND** Boolean return is false

### Requirement: ExtractCostInUsdTicksFromSSEChunk

The system SHALL extract cost in usd ticks from an sse chunk using top-level usage.cost_in_usd_ticks field used by grok cli.

#### Scenario: ValidSSEChunkWithGrokCLICost
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** An SSE chunk containing usage.cost_in_usd_ticks field
- **WHEN** ExtractCostInUsdTicksFromSSEChunk is called
- **THEN** Returns cost value divided by grokCLICostUsdTicksDivisor and true
- **AND** Boolean return is true

#### Scenario: SSEChunkWithoutGrokCLICost
- **GIVEN** An SSE chunk without usage.cost_in_usd_ticks field
- **WHEN** ExtractCostInUsdTicksFromSSEChunk is called
- **THEN** Returns zero and false
- **AND** Boolean return is false

### Requirement: MergeTokenCounts

The system SHALL merge source token counts into destination, where incoming non-zero values overwrite target values, matching omniroute's mergeusage pattern.

#### Scenario: MergeNonZeroInputTokens
<!-- openlore-test: priority=high (auto) -->
- **GIVEN** Destination StreamTokenCounts with existing values and source with non-zero InputTokens
- **WHEN** MergeTokenCounts is called
- **THEN** Destination is updated with source's non-zero values
- **AND** dst.InputTokens updated to src.InputTokens
- **AND** Other dst fields remain unchanged if src fields are zero

#### Scenario: MergeAllZeroSource
- **GIVEN** Destination StreamTokenCounts with existing values and source with all zero values
- **WHEN** MergeTokenCounts is called
- **THEN** Destination remains unchanged
- **AND** All fields in dst remain unchanged

#### Scenario: CompleteMergeOverwrite
- **GIVEN** Destination with partial values and source with all non-zero values
- **WHEN** MergeTokenCounts is called
- **THEN** Destination is fully updated with all source values
- **AND** dst.InputTokens, OutputTokens, ReasoningTokens, CachedTokens, CacheCreationTokens, and CostUsd all
updated

## Sub-components

> `DashboardHandler` is an orchestrator. Each sub-component below implements one logical block.

### Sub-component: RowsClose

> Implements: `rows.Close`

Releases database resources held by an active query result set to prevent connection leaks.

#### Requirement: CloseResultSet

The system SHALL the rows.close operation shall release all resources associated with the active database result set.

#### Scenario: CloseActiveResultSet
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** a database query has returned an active rows cursor with pending results
- **WHEN** rows.Close is invoked on the active cursor
- **THEN** all resources associated with the rows cursor are released and the cursor becomes invalid
- **AND** no subsequent operations on the closed rows SHALL succeed

### Sub-component: RowsNext

> Implements: `rows.Next`

Advances the database cursor through the result set row by row for sequential processing.

#### Requirement: AdvanceCursor

The system SHALL the rows.next operation shall advance the cursor to the next row in the result set and report whether a row is available.

#### Scenario: AdvanceToNextRow
- **GIVEN** a database query has returned a rows cursor with multiple result rows
- **WHEN** rows.Next is called and additional rows remain in the result set
- **THEN** the cursor advances to the next row and returns true
- **AND** subsequent rows.Scan SHALL populate values from the current row

#### Scenario: NoMoreRows
- **GIVEN** a database query has returned a rows cursor and all rows have been iterated
- **WHEN** rows.Next is called after the final row has been consumed
- **THEN** rows.Next returns false indicating no further rows are available
- **AND** rows.Close SHOULD be called after iteration completes

### Sub-component: RowsScan

> Implements: `rows.Scan`

Extracts column values from the current database row into Go variables for application use.

#### Requirement: ScanRowValues

The system SHALL the rows.scan operation shall copy column values from the current row into the provided destination variables.

#### Scenario: ScanValidRow
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** rows.Next has returned true and destination pointers are provided for all columns
- **WHEN** rows.Scan is called with properly typed destination pointers
- **THEN** column values are copied into the destination variables and nil error is returned
- **AND** the number of destination arguments MUST match the number of columns in the query

#### Scenario: ScanTypeMismatch
<!-- openlore-test: tags=regression (auto) -->
- **GIVEN** rows.Next has returned true but destination pointer type does not match column type
- **WHEN** rows.Scan is called with a destination pointer of incompatible type
- **THEN** an error is returned and destination variables remain unmodified
- **AND** the current row position MUST NOT advance automatically
